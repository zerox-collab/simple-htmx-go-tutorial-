Exercise 1: Click to Change Text

🎯 Core Concept: Swapping an Element

The most basic HTMX action. When you click the button, it asks the server for new HTML and replaces itself with the response.

Key Learning Points:
  • hx-post: Sends a request to the server.
  • hx-swap="outerHTML": Replaces the entire element (the button itself) with the HTML from the server's response.

Exercise 2: Click to Load Content

🎯 Core Concept: Targeting a Different Element

This shows how to make one element (a button) load content into a different element (a div).

Key Learning Points:
  • hx-get: Sends a request when the button is clicked.
  • hx-target: A CSS selector (`#ex2-target`) tells HTMX where to put the response. The default swap strategy is `innerHTML`.

Exercise 3: Polling for Updates

🎯 Core Concept: Timed Triggers

HTMX isn't just for clicks. It can trigger requests on a timer, perfect for live data like clocks, notifications, or dashboards.

Key Learning Points:
  • hx-trigger="every 2s": This modifier tells HTMX to send a GET request to the specified URL every 2 seconds.
  • The element will update itself with the response automatically.

Exercise 4: Send User Input

🎯 Core Concept: Event-Driven Requests

Trigger requests from any event, like typing into an input. Debouncing prevents sending too many requests.

Key Learning Points:
  • hx-trigger="keyup changed delay:500ms": Send a request on the `keyup` event, but only if the value `changed`, and wait `500ms` after the user stops typing.
  • The input's `name` and `value` are automatically sent as URL parameters.

Exercise 5: Form Submission & Loading Indicators

🎯 Core Concept: Form Handling & UX Feedback

Handle form submissions without a page reload and provide visual feedback with loading indicators.

Key Learning Points:
  • hx-post: Serializes the form data and sends it in a POST request.
  • hx-indicator: A CSS selector for an element to show while the request is in flight.

Exercise 6: Click To Edit

🎯 Core Concept: In-Place Editing

A common web pattern where the server sends a form to edit data, and then replaces the form with the updated view upon submission.

Key Learning Points:
  • State Transitions: The server controls the UI by sending back either the 'view' or 'edit' template.
  • HTTP Methods: Use `GET` to request the edit form and `PUT` (or POST) to submit the update.
